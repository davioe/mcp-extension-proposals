# SEP Submission Checklist

Steps to submit a Specification Enhancement Proposal to the official MCP repository.

## Pre-Submission

1. Post in MCP Discord `#ideas` channel to gauge community interest
2. Gather initial feedback and refine the proposal

## Submission

3. Fork `modelcontextprotocol/modelcontextprotocol` on GitHub
4. Copy your SEP file to `seps/0000-your-title.md` in the fork
5. Open a Pull Request to the `main` branch
6. Note the PR number — this becomes your SEP number
7. Amend your commit: rename the file to `{PR_NUMBER}-your-title.md`
8. Update the `SEP Number` field in the preamble with the PR number
9. Update the `Discussion` field with the PR URL

## Review

10. Request a Sponsor (Core Maintainer or Maintainer) by commenting on the PR or reaching out on Discord
11. Iterate based on review feedback
12. Sponsor manages status transitions: Draft → In-Review → Accepted → Final
